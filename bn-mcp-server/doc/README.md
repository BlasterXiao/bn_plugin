## 与 `spec/` 目录的区别

| 目录 | 读者 | 内容 |
|------|------|------|
| **`doc/`**（此处） | 插件用户、逆向分析使用者 | 使用说明、场景与能力介绍 |
| **`spec/`** | 维护者、贡献者、产品/研发 | 需求文档、开发计划、技术实现指南等内部规格材料 |

`spec/` 放在仓库根下是常见做法（与实现代码并列，便于版本管理），**无需改名**。若你希望强调「仅内部阅读」，也可在团队约定中把 `spec` 视为设计文档区；从目录语义上，`spec` 已能表达「规格/说明文书」的含义。
