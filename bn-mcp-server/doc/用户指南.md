# Binary Ninja MCP Server — 用户指南

## 插件是什么

**Binary Ninja MCP Server** 是运行在 **Binary Ninja 图形界面进程内** 的一款插件。它在本地开启一个符合 **Model Context Protocol（MCP）** 的 HTTP 服务，让 **Cursor** 等 AI 客户端能够通过标准协议调用 Binary Ninja 的分析能力（反汇编、反编译、交叉引用、符号与类型、补丁等），而无需在无头模式下单独启动 Binary Ninja。

适用人群：

- 已使用 **Binary Ninja** 做逆向、漏洞分析或二进制研究的分析师；
- 希望在 **AI 辅助** 下完成「查函数、读 HLIL、跟 xref、改注释/重命名」等重复或探索性工作的用户。

---

## 能为你做什么

在典型工作流中，你可以在 Cursor 里让 AI：

- **浏览当前二进制**：段信息、入口点、已加载视图等；
- **阅读与分析代码**：函数列表、HLIL/LLIL/MLIL、控制流图（CFG）相关查询；
- **导航与关联**：交叉引用（xref）、符号、字符串；
- **类型与元数据**：类型定义、重命名、注释（在权限与线程模型允许的情况下）；
- **修改与自动化**：补丁、分析选项控制，以及插件提供的进阶辅助能力。

具体工具名称与参数以客户端中列出的 MCP **Tools** 为准；仓库 README 中「功能概览」亦给出了能力范围说明。

## 使用前请知悉

1. **在 GUI 中运行**  
   插件依赖 Binary Ninja 已加载的 Python 环境与当前打开的 **`BinaryView`**。适合 **Binary Ninja Pro** 等无法在独立进程里 `import binaryninja` 做无头自动化的场景。

2. **先打开样本再启动服务（推荐）**  
   建议在已打开目标二进制并确认分析就绪后，再通过菜单启动 MCP，以便正确绑定当前活动视图。若遇「无 BinaryView」类提示，可先切换标签或按 README 中的排错步骤操作。

3. **网络与认证**  
   服务默认监听本机地址（具体端口见 **Plugins → MCP Server → Status**）。访问需使用 Status 中提供的 **Bearer Token**；请勿把 Token 提交到公共仓库或发给不可信方。

4. **数据边界**  
   AI 客户端只会通过你配置的地址访问本机服务；二进制内容是否进入模型上下文取决于客户端与对话设置。敏感样本请遵守本单位保密规定。

## 安装与启动（摘要）

1. 将 **`bn-mcp-server`** 整个文件夹放入 Binary Ninja **用户插件目录**（各操作系统路径见根目录 README）。
2. 在 Binary Ninja 使用的 **同一 Python 环境** 中安装 `requirements.txt` 中的依赖。
3. 重启 Binary Ninja，菜单 **Plugins → MCP Server → Start** 启动服务；**Stop** 停止；**Settings** 调整端口等；**Status** 查看 URL 与 Token。

完整命令、依赖说明与常见问题见 [README.md](../README.md)。

---

## 连接 Cursor（摘要）

在 Cursor 的 MCP 配置中使用 **Streamable HTTP** 地址，例如：

- URL：`http://127.0.0.1:<端口>/mcp`（路径需包含 **`/mcp`**）
- 请求头：`Authorization: Bearer <与 Status 中一致的 Token>`

Cursor 可能触发浏览器 OAuth 流程；插件已做最小兼容实现，完成后获得的令牌与 Status 中的预共享 Token 一致。更详细的 OAuth 说明、404 与 `unsupported_response_type` 等排错见根目录 README。

## 菜单项说明

| 菜单项       | 作用                                                            |
| ------------ | --------------------------------------------------------------- |
| **Start**    | 启动 MCP HTTP 服务；已在运行时通常会置灰，避免重复启动。        |
| **Stop**     | 停止服务。                                                      |
| **Settings** | 配置监听地址、端口等（具体项以插件界面为准）。                  |
| **Status**   | 查看服务是否运行、**复制 Base URL 与 Token**，用于配置 Cursor。 |

## 能力概览（按模块）

以下为能力分组，便于理解；实际以 MCP 工具列表为准。

| 模块                  | 典型用途                              |
| --------------------- | ------------------------------------- |
| **binary_view**       | 当前视图信息、读取字节、导航元数据等  |
| **functions**         | 函数枚举、按名/地址查询、各层 IL 文本 |
| **xrefs**             | 引用与被引用关系                      |
| **symbols / strings** | 符号表与字符串检索                    |
| **types**             | 类型查询与部分类型操作                |
| **edit**              | 注释、重命名等编辑类操作              |
| **cfg**               | 控制流相关查询                        |
| **patch**             | 二进制补丁                            |
| **analysis**          | 分析进度与选项控制                    |
| **advanced**          | 插件提供的进阶/启发式辅助能力         |
